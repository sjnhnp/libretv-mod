# Detail特殊数据源处理修复说明

## 修复内容

### 1. 增强API站点配置
- 为所有主要API站点添加了`detail`字段配置
- 新增了飞飞资源(ffzy)用于测试特殊格式

### 2. 增强正则表达式匹配
- 添加了`M3U8_PATTERN_ENHANCED`：支持更多m3u8格式
- 添加了`VIDEO_URL_PATTERN`：支持多种视频格式(.mp4, .flv, .avi, .mkv, .ts)

### 3. 改进特殊源处理逻辑
- `handleSpecialSourceDetail`函数增强：
  - 支持多种匹配策略（特殊格式 → 增强匹配 → 原始匹配 → 广泛匹配）
  - 增加播放列表提取逻辑
  - 改进标题和描述提取
  - 添加详细日志输出

### 4. 改进自定义API处理
- `handleCustomApiSpecialDetail`函数增强：
  - 支持多层级匹配策略
  - 增加播放列表提取逻辑
  - 传递自定义API信息用于更好的源名称显示

### 5. 优化API请求处理流程
- 改进detail处理的优先级逻辑：
  1. 优先使用配置了detail的内置源进行HTML抓取
  2. 自定义源支持detail配置或强制HTML抓取
  3. 标准JSON API作为回退选项
  4. 如果标准API失败且有detail配置，自动回退到HTML抓取

### 6. 增强自定义API管理
- 添加详情页URL输入框支持
- 更新添加、编辑、更新逻辑以支持detail字段
- 改进表单重置逻辑

## 解决的问题

1. **detail特殊数据源播放页无法播放**
   - 通过多层级匹配策略确保能提取到有效播放链接
   - 增加回退机制，提高成功率

2. **自定义API源的detail处理缺失**
   - 添加detail字段支持
   - 实现HTML抓取逻辑

3. **播放链接提取不完整**
   - 增强正则表达式匹配
   - 添加播放列表提取逻辑

4. **错误处理和日志不足**
   - 添加详细的控制台日志
   - 改进错误信息传递

## 使用方法

### 内置源
配置了detail字段的内置源会自动使用HTML抓取：
- 黑木耳、电影天堂、魔爪资源、如意资源等

### 自定义源
1. 添加自定义API时，可以填写"详情页URL"字段
2. 如果不填写，系统会尝试标准JSON API
3. 如果填写了detail URL，会优先使用HTML抓取

### 调试
- 打开浏览器开发者工具查看控制台日志
- 搜索`[特殊源详情]`或`[自定义源详情]`查看处理过程

## 测试建议

1. 测试内置源（如黑木耳、电影天堂）的播放功能
2. 添加自定义API并测试detail功能
3. 检查控制台日志确认处理流程
4. 验证播放链接提取是否成功
